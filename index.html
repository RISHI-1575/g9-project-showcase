Privacy & Security · LLM / NLP
AEGIS-X
Adaptive, context-aware guard for information security in generative AI
NLP Adversarial ML Transformer fine-tuning Differential privacy Python HuggingFace FastAPI Streamlit
A smart privacy layer that sits between you and any LLM. It reads your prompt, understands context (not just keywords), transforms secrets into safe equivalents, sends the clean version to ChatGPT, and restores real values in the answer — so you get a useful response without leaking anything.
How it works
  • 1User types a prompt containing sensitive data
  • 2Fine-tuned DeBERTa reads full context — not just keywords — and labels each span: public / PII / confidential
  • 3Sensitive spans are transformed, not blanked — “Project Bluebird  ₹12Cr” → “an internal initiative”
  • 4Adversarial attacker network tries to reverse-engineer secrets; sanitizer trains to beat it (min-max SGD)
  • 5Clean prompt reaches the LLM; answer comes back and real values are restored via encrypted local vault
  • 6User gets a full, usable answer. The LLM never saw the secret.

What makes AEGIS-X different
  • Adversarially trained sanitizer — a second network attacks it during training, forcing genuine leakage resistance
  • Formal differential-privacy bound: measurable guarantee of what the LLM provider can learn
  • Indian-context data: Aadhaar, PAN, Hinglish, Indian business terminology — no product covers this
  • Utility-preserving transformation (not blanking) — LLM still gives a full, useful answer
  • Fully open, reproducible benchmark released publicly — something commercial tools refuse to do
AI Security · Offensive + Defensive
Chakravyuh-2
Co-evolutionary AI red-team agent that hacks LLM systems — and builds the fix
Reinforcement learning Self-play LLM agents Mechanistic interpretability LangGraph PyTorch / TRL Ollama Python
An autonomous system that attacks AI apps the way a real hacker would — then learns, adapts, and patches the defense. A multi-agent pipeline where an attacker and defender co-evolve via self-play until the system is provably hardened, with mechanistic explanations for every vulnerability found.
How it works
  • 1Recon agent profiles the target AI app — what tools it has, what secrets it guards
  • 2RL attacker generates adversarial prompts; learns from success/failure via PPO reward signal
  • 3Judge agent evaluates: did the target leak a secret? misuse a tool? violate policy?
  • 4Attacker and defender train against each other in self-play — like AlphaGo vs itself — until equilibrium
  • 5Interpretability layer probes model activations to explain why each jailbreak works
  • 6Reporter auto-generates an OWASP-mapped PDF audit with patch suggestions

What makes Chakravyuh-2 different

  • Open reproducible benchmark for attacker/defender pairs — no standard one exists today
Smart Cities · Reinforcement Learning
Saarathi
Equity-aware, fault-tolerant, learned-communication traffic brain for Indian cities
Multi-agent RL Emergent communication Graph attention networks Multi-objective optimization SUMO simulator PyTorch Ray RLlib Python
Every traffic junction is an autonomous agent. Agents don’t just coordinate — they learn what to say to each other, optimize for wait time, emissions, ambulance corridors, and neighborhood fairness simultaneously, and keep working even when junctions go offline or get compromised.
How it works
  • 1Each junction runs its own RL agent watching real-time traffic: vehicle count, queue length, speed
  • 2Agents learn what to communicate via graph-attention differentiable messaging — not hard-coded outflow numbers
  • 3Multi-objective reward: minimize wait + cut emissions + green corridor for ambulances + fairness score across zones
  • 4Byzantine fault tolerance: if a hub fails or is tampered with, neighbors self-compensate seamlessly
  • 5Trained on SUMO with real Bengaluru junction topology: lane-less, mixed Indian traffic (bikes, autos, trucks)
  • 6Benchmarked vs fixed-time signals + SURTRAC-style coordination on all metrics simultaneously

What makes Saarathi different
  • Learned inter-agent communication — hubs discover what to signal, not told (emergent protocol via graph-attention)
  • Equity objective: explicitly prevents congestion from being pushed onto lower-income neighborhoods
  • Byzantine resilience: stays operational when hubs fail or are hacked — no existing traffic MARL paper tests this
  • Indian heterogeneous traffic: lane-less, mixed vehicle types — absent from all existing products and papers
  • Multi-objective Pareto frontier: quantified trade-off curve between delay, emissions, and equity — publishable
